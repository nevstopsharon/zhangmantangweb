# 代码风格指南

## JavaScript
- 使用 ES6+ 语法
- 采用模块化设计
- 函数和变量命名使用驼峰命名法
- 常量使用大写蛇形命名法
- 缩进使用 2 个空格
- 每行代码不超过 80 字符
- 使用单引号
- 添加适当的注释

## CSS
- 使用 CSS 变量管理设计系统
- 类名使用小写连字符命名法
- 缩进使用 2 个空格
- 每行代码不超过 80 字符
- 遵循 BEM 命名规范
- 添加适当的注释

## HTML
- 使用语义化标签
- 缩进使用 2 个空格
- 属性值使用双引号
- 保持标签闭合
- 添加适当的注释

## 注释规范
- 函数注释：描述功能、参数、返回值
- 代码块注释：描述逻辑和实现细节
- 关键变量注释：解释变量用途
- 文件头部注释：描述文件功能和结构

## 命名规范
- 文件名：小写连字符命名法
- 目录名：小写连字符命名法
- 函数名：驼峰命名法
- 变量名：驼峰命名法
- 常量名：大写蛇形命名法
- 类名：帕斯卡命名法

## 最佳实践
- 保持代码简洁明了
- 避免重复代码
- 优先使用函数式编程
- 注重性能优化
- 确保代码可维护性
