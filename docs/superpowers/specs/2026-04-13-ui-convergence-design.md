# 张满堂官网中度收敛版界面优化设计

日期：2026-04-13

## 目标

在不改数据结构、不重做交互框架的前提下，把当前网站界面从“信息系统感”收敛到更稳定的“展陈阅读感”。

本轮只处理四件事：

1. 作品详情页删除列表页遗留筛选器。
2. 展览与新闻的年份轨道强化当前态与点击热区。
3. 内页联系区改为紧凑版，首页保留完整版本。
4. 去掉页面左右限制线，统一页面背景为导航栏同色背景。

## 约束

- 不改 `data/*.json` 结构。
- 不改 `excel/content.xlsx`。
- 不改搜索逻辑、详情页媒体切换逻辑、年份点回跳逻辑。
- 不改整体色彩主体系与字体体系。
- 改动必须同步 design system 文档与设计令牌。

## 当前问题

### 1. 作品详情页混入列表页工具控件

从第一性原理看，详情页的唯一主角应当是“作品本身”。当前作品详情页顶部仍保留“全部 / 项目 / 材质”这类筛选控件，会把用户从作品阅读状态拉回列表管理状态。

### 2. 年份轨道功能已存在，但视觉上不像导航

当前年份轨道可点击、也能回跳，但非当前态过轻，当前态强调不足，导致用户更容易把它理解成辅助刻度，而不是主导航。

### 3. 联系区在内页收尾阶段过重

首页的完整联系区合理，但同样体量落到作品、展览、新闻详情页底部时，会打断内容阅读收束，造成“刚读完正文就被新大模块接管”的感受。

### 4. 左右边界线把页面气质拉向“信息框架”

当前基础规范依赖左右弱边框建立秩序，这种方式有效，但会强化界面的系统边框感。对于当前网站，以统一背景和留白建立秩序，更符合内容展陈目标。

## 方案选择

### 方案 A：保守微调

只做局部样式调整，不改 design system 基线。

优点：
- 风险最低
- 改动最小

缺点：
- 无法解决边界线和内页联系区的系统性问题
- 设计语言仍然前后冲突

### 方案 B：中度收敛版（采用）

保留当前色彩、字体、留白体系，但把空间组织方式从“边框定义空间”调整为“背景与留白定义空间”，同时引入内页联系区紧凑变体。

优点：
- 收益明显
- 改动仍可控
- 与当前站点已有气质兼容

缺点：
- 需要同步更新 design system 基线

### 方案 C：强展陈改版

在方案 B 基础上，连首页首屏比例、列表卡片尺度、详情页标题层级一并重做。

优点：
- 视觉提升最大

缺点：
- 超出本轮范围
- 验证成本明显上升

## 采用方案

采用方案 B：中度收敛版。

## 设计更新

### 一、基础规范更新

#### 1. 页面背景

- 全站页面背景统一使用当前导航栏背景色，即基础背景色 `#F2EFED`。
- 页面秩序主要依赖：
  - 内容容器宽度
  - 区块留白
  - 标题层级
  - 轻分隔

#### 2. 页面边界

- 去掉页面左右两条限制线。
- 保留内容最大宽度控制。
- 不再把左右边框作为“展陈边界”的主要手段。

#### 3. 仍保留的秩序工具

- 大标题
- 暖金强调色
- 中灰辅助信息
- 区块间距
- 局部分隔线

### 二、组件规范更新

#### 1. 作品详情页顶部工具区

当前详情页中出现的：

- 全部
- 项目
- 材质

全部移除。

作品详情页顶部只保留：

- 返回入口
- 页面标题
- 主图区域
- 作品信息区

原则：
- 筛选属于列表页，不属于详情页。
- 详情页不得混入列表页筛选行为入口。

#### 2. 年份轨道

年份轨道的角色从“辅助刻度”提升为“显式导航”。

更新方向：

- 当前年份文字颜色加深。
- 当前年份增加更明显的暖金强调。
- 每个年份按钮的垂直点击热区变大。
- hover、active、current 三种状态明确区分。
- 非当前年份仍保持克制，但不能看起来像禁用态。

当前态应传达两件事：

1. 我现在在哪个年份。
2. 这一列是可以点击导航的。

#### 3. 联系模块变体

联系模块从单一版本改为两个变体：

##### 首页完整联系模块

保留当前结构：
- 左侧大标题
- 右侧完整联系信息
- 更充足的上下留白

##### 内页紧凑联系模块

适用于：
- 作品详情页
- 展览详情页
- 新闻详情页

收紧方向：
- 模块高度减少
- 标题字号下调一级
- 标签字重更轻
- 行距与上下边距更紧
- 保留全部联系信息，不删字段

原则：
- 首页负责建立合作入口。
- 内页负责完成阅读收尾，不抢正文结尾的主导权。

### 三、页面模式更新

#### 1. 首页

- 保留完整联系模块。
- 背景统一为基础背景色。
- 去掉左右限制线后，首页靠留白与区块节奏建立秩序。

#### 2. 作品详情页

- 删除筛选器。
- 使用紧凑联系模块。
- 主图与信息区继续保持现有结构。

#### 3. 展览详情页 / 新闻详情页

- 年份轨道强化当前态与点击热区。
- 使用紧凑联系模块。
- 保持年份点回总页并滚动到对应分组的现有逻辑。

### 四、设计令牌更新

本轮不重做主色体系，只增改以下令牌类别：

#### 1. 边界相关

- 废止“页面左右限制线”为页面主秩序工具的表述。
- 保留局部分隔线和弱边框能力，用于卡片或局部模块。

#### 2. 年份轨道

新增或更新：
- 当前年份颜色
- 当前年份强调标记尺寸
- 年份按钮垂直内边距
- 年份按钮最小点击高度

#### 3. 紧凑联系区

新增或更新：
- 内页联系区上下边距
- 内页联系区标题字号
- 内页联系区行距
- 标签列与内容列间距

- `site/js/renderers.js`
- `site/js/components.js`

但原则上优先只用样式和局部模板调整完成。

## 不在本轮范围内

- 首页首屏比例调整
- 作品列表卡片尺度重做
- 搜索组件视觉重做
- 详情页标题层级重做
- 移动端单独设计

## 验收标准

### 视觉验收

1. 作品详情页顶部不再出现“全部 / 项目 / 材质”。
2. 展览与新闻详情页中，年份轨道当前态能一眼被识别。
3. 内页联系区明显比首页联系区更紧凑。
4. 页面左右限制线消失后，界面不显得散乱，背景统一。

### 结构验收

1. design system 文档与页面实现一致。
2. 不出现“文档仍说全站统一联系模块，但代码已分为两种变体”的冲突。
3. 不影响现有详情页媒体切换和年份回跳行为。

## 实施顺序

1. 更新 design system 文档与设计令牌。
2. 落地页面样式与局部模板调整。
3. 跑浏览器清单复验：
   - 中文首页
   - 作品详情页
   - 展览详情页
   - 新闻详情页
4. 如有必要，再做二次细调。
