# 00 治理规则

## 1. 文档边界
本设计系统只覆盖“当前官网前端实际实现”，不覆盖：
- 商业 PRD
- 投资人文档
- 履历原始资料
- 线下印刷物规范

## 2. 当前真实来源
设计系统信息应以以下文件为准：
- 视觉与组件：`site/css/style.css`
- 页面结构与交互：`site/js/main.js`
- 站点级品牌与 about/contact 内容：`data/profile.json`

## 3. 变更顺序
1. 先确认页面改动是否会影响全站一致性。
2. 若影响颜色、字号、间距、断点或组件形态，先更新设计系统文档。
3. 再修改前端代码。
4. 修改后做浏览器验收，并在 `CHANGELOG.md` 记录变更。

## 4. 版本策略
- `Major`：视觉语言或页面结构大改
- `Minor`：新增组件或显著调整既有组件规则
- `Patch`：对齐代码与文档、修复错漏、细节微调

当前版本：`1.2.0`

## 5. 验收标准
一次有效的设计系统更新，至少要满足：
- 令牌值与 CSS 一致
- 页面模式与当前站点一致
- 组件说明能解释当前界面长什么样、为什么这样排
- 内容规范能指导 Excel / JSON / 图片目录的协作方式
